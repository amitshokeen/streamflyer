017package com.github.rwitzel.streamflyer.core;
018
019/**
020 * A modifier replaces, deletes, inserts characters in a character stream. This happens in interaction with a
021 * <em>stream processor</em>, i. e. either a {@link ModifyingReader} or a {@link ModifyingWriter}. The modifier can be
022 * considered a callback that is called by the stream processor to make the desired modifications. The pivotal part of
023 * the interaction between the modifier and the stream processor is a character buffer the stream processor facilitates.
024 * That buffer contains the characters of the underlying character stream.
025 * <p>
026 * Besides the modifications of the buffer, the modifier manages the capacity (size) of the character buffer and
027 * {@link AfterModification defines} what the stream processor shall do before the stream processor
028 * {@link Modifier#modify(StringBuilder, int, boolean) calls} the modifier the next time.
029 * <p>
030 * Implementations of this interface should inform the user about its specific memory consumption because the most
031 * important feature of modifiers should be their small memory foot-print.
032 * 
033 * @author rwoo
034 * @since 06.05.2011
035 */
036public interface Modifier {
037
038    /**
039     * Processes the characters in the given character buffer, i.e. deletes or replaces or inserts characters, or keeps
040     * the characters as they are.
041     * 
042     * @param characterBuffer
043     *            The next characters provided from the modifiable stream. It contains the modifiable characters,
044     *            (optionally) preceded by unmodifiable characters.
045     *            <p>
046     *            The modifier can modify the content of the buffer as appropriate the modifier must not modify the
047     *            unmodifiable characters, i.e. the characters before position
048     *            <code>firstModifiableCharacterInBufferIndex</code> must not be modified.
049     *            <p>
050     *            Your modifier should manage the {@link StringBuilder#capacity() capacity} of the buffer on its own -
051     *            as the optimal management of the capacity depends on the specific purpose of the modifier.
052     *            <p>
053     *            The given buffer must be never null.
054     * @param firstModifiableCharacterInBuffer
055     *            index of the first modifiable character in the buffer. The index of the first character in the buffer
056     *            is zero. If there is not modifiable character in the buffer, then this value is equal the length of
057     *            the buffer.
058     * @param endOfStreamHit
059     *            True if no more characters can be read from the stream, i.e. the character buffer contains the all the
060     *            characters up to the end of the stream.
061     * @return Returns an object that defines how {@link ModifyingReader} or {@link ModifyingWriter} shall behave before
062     *         calling {@link Modifier#modify(StringBuilder, int, boolean)} again.
063     */
064    public AfterModification modify(StringBuilder characterBuffer, int firstModifiableCharacterInBuffer,
065            boolean endOfStreamHit);
066}
